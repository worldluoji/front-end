# 油猴脚本
Tampermonkey 是目前最流行的用户脚本管理器，可以让你通过运行自定义 JavaScript 脚本来增强网页功能。下面是它的完整使用指南，涵盖安装、获取脚本、自己编写脚本、管理及调试。

---

### 一、安装 Tampermonkey

根据你的浏览器，在对应扩展商店搜索 **Tampermonkey** 安装：

- **Chrome / Edge**：[Chrome 网上应用店](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
- **Firefox**：[Firefox 附加组件](https://addons.mozilla.org/firefox/addon/tampermonkey/)
- **Safari**：需要从官网下载桌面版（部分版本可能收费）

安装后，浏览器右上角会出现 Tampermonkey 图标（一个带有两个圆点的猴子面具）。

---

### 二、安装别人写好的脚本

1. 打开用户脚本仓库，最常用的是 **GreasyFork**（https://greasyfork.org/）。
2. 搜索你需要的功能，例如“百度网盘直链”“视频网站去广告”等。
3. 进入脚本详情页，点击 **“安装此脚本”**。
4. Tampermonkey 会弹出安装确认界面，再次点击 **“安装”** 即可。

> **内网专用脚本**：如果公司内部有分享的 `.user.js` 文件，可以直接在浏览器打开该文件，Tampermonkey 会自动识别并提示安装。

---

### 三、自己编写脚本（重点）

以之前讨论的“内网重复表单填充”为例，演示如何创建并编写脚本。

#### 1. 创建新脚本
- 点击 Tampermonkey 图标 → **“创建新脚本”**。
- 编辑器会打开，默认包含一个脚本模板。

#### 2. 理解脚本结构
```javascript
// ==UserScript==
// @name         我的内网自动填表
// @namespace    http://tampermonkey.net/
// @version      0.1
// @description  自动填充内网表单
// @author       You
// @match        *://your-internal-domain.com/*
// @grant        none
// ==/UserScript==

(function() {
    'use strict';

    // 你的代码写在这里
})();
```

- **`@name`**：脚本名称，在管理面板显示。
- **`@match`**：**最关键**，指定脚本在哪些网址生效。支持通配符：  
  - `*://*/*` 匹配所有网址（不推荐）  
  - `*://your-domain.com/*` 匹配该域名下所有页面  
  - `https://your-domain.com/form/*` 精确匹配特定路径  
  针对内网 URL 变化但域名不变的情况，使用 `*://your-domain.com/*` 即可。
- **`@grant`**：声明需要使用的特殊 API，如 `GM_setValue`（存储数据）、`GM_xmlhttpRequest`（跨域请求）。若只操作 DOM，写 `none` 即可。

#### 3. 编写填充逻辑
根据内网表单的特征，使用 DOM 操作填入内容。需要注意单页应用（SPA）的页面切换问题，使用 `MutationObserver` 等待目标元素出现。

**示例脚本（自动填充姓名、部门）**：
```javascript
// ==UserScript==
// @name         内网表单自动填充
// @match        *://your-internal.com/*
// @grant        none
// ==/UserScript==

(function() {
    'use strict';

    // 要填充的数据
    const data = {
        name: '张三',
        department: '技术部'
    };

    // 等待表单出现的函数
    function waitForForm() {
        const observer = new MutationObserver((mutations, obs) => {
            // 尝试找到关键输入框（根据实际页面特征修改选择器）
            const nameInput = document.querySelector('#user_name');
            if (nameInput && !nameInput._filled) {
                // 填充
                nameInput.value = data.name;
                nameInput.dispatchEvent(new Event('input', { bubbles: true }));

                const deptSelect = document.querySelector('#department');
                if (deptSelect) {
                    deptSelect.value = data.department;
                    deptSelect.dispatchEvent(new Event('change', { bubbles: true }));
                }

                // 标记已填充，避免重复
                nameInput._filled = true;
                console.log('表单已自动填充');
                obs.disconnect(); // 停止观察
            }
        });

        observer.observe(document.body, { childList: true, subtree: true });
    }

    // 页面可能未加载完，先等DOM就绪
    if (document.readyState === 'loading') {
        document.addEventListener('DOMContentLoaded', waitForForm);
    } else {
        waitForForm();
    }
})();
```

#### 4. 保存脚本
- 点击编辑器左上角的 **“文件”** → **“保存”**（或按 `Ctrl+S`）。
- 脚本即开始生效，下次访问匹配的页面时会自动运行。

---

### 四、脚本管理

- **启用/禁用**：点击 Tampermonkey 图标，在弹出的面板中，每个脚本旁边都有一个开关按钮，可随时启用或禁用。
- **编辑/删除**：点击图标 → **“管理面板”**，进入仪表盘。在脚本列表中点击“编辑”修改代码，或点击“垃圾桶”删除。
- **备份与导出**：在管理面板的“实用工具”选项卡中，可以导出所有脚本为压缩包，或导入备份。

---

### 五、调试技巧

当脚本不生效时，可以这样排查：

1. **查看控制台日志**  
   在页面按 `F12` 打开开发者工具，切换到 **Console** 面板。如果你在脚本中写了 `console.log()`，会在这里输出。错误信息也会显示。

2. **检查 @match 是否匹配**  
   在管理面板中找到脚本，查看“包括/排除”规则，确认当前页面的 URL 是否在匹配范围内。也可以临时将 `@match` 改为 `*://*/*` 测试，但正式使用时务必收窄范围。

3. **使用 `@run-at` 控制执行时机**  
   若脚本需要尽早执行，可在元数据中添加 `// @run-at document-start`，这样脚本会在 DOM 构建前注入，适用于拦截 XHR 请求或监听早期事件。

4. **单页应用（SPA）的页面切换**  
   如之前所述，使用 `MutationObserver` 监听 DOM 变化，避免因页面切换导致脚本只执行一次而失效。

---

### 六、安全与合规提醒

- **内网使用**：在内网编写脚本时，注意不要将公司内部系统的选择器、敏感逻辑发布到公共脚本网站。
- **避免冲突**：如果内网系统本身已有 JavaScript 全局变量，你的脚本可能造成冲突，使用 IIFE 或 `let` 声明局部变量即可。
- **性能**：`MutationObserver` 记得在完成任务后调用 `disconnect()`，避免长期运行消耗资源。
