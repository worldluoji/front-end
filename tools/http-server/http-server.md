## http-server 使用方法与场景

`http-server` 是一个零配置、开箱即用的命令行静态 HTTP 服务器，基于 Node.js 开发。非常适合快速预览静态项目或进行本地开发调试。

### 一、安装

全局安装（推荐）：
```bash
npm install -g http-server
```

项目内安装：
```bash
npm install --save-dev http-server
```

### 二、基本使用

进入目标目录（如项目根目录），运行：
```bash
http-server
```

默认启动在 `http://localhost:8080`，同时显示网络中的 IP 地址（如 `http://192.168.1.100:8080`）。  
浏览器访问即可看到目录文件列表（默认启用目录浏览）。

### 三、常用选项

| 选项 | 说明 |
|------|------|
| `-p <端口>` | 指定端口，例如 `-p 3000` |
| `-a <地址>` | 绑定地址，默认 `0.0.0.0`（所有网络接口） |
| `-d` | 显示目录列表（默认开启），禁用可用 `-d false` |
| `-o` | 启动后自动打开浏览器 |
| `-c <秒数>` | 设置 Cache-Control 的 max-age，默认 `3600`（1小时） |
| `--cors` | 启用跨域资源共享（CORS），允许所有来源 |
| `-S` / `-C` | 启用 HTTPS，需提供证书文件（默认自签名） |
| `-g` / `-G` | 启用 Gzip 压缩 |
| `-b` | 禁止浏览器自动打开（配合 `-o` 使用时无效） |
| `--proxy <url>` | 将匹配的请求代理到目标 URL（简单代理） |
| `-h` | 显示帮助 |

示例：
```bash
# 监听 3000 端口，自动打开浏览器，启用 CORS
http-server -p 3000 -o --cors

# 启动 HTTPS 服务（自动生成证书）
http-server -S -p 443

# 禁用目录列表，缓存设为 0（不缓存）
http-server -d false -c 0
```

### 四、典型使用场景

1. **快速预览静态页面**  
   开发 HTML/CSS/JS 原型时，直接 `http-server` 即可避免 `file://` 协议带来的资源加载限制（如无法使用 Fetch、模块导入等）。

2. **解决跨域问题**  
   前端代码需要请求本地 JSON 或其他资源，若用 `file://` 会被浏览器跨域策略阻挡。用 `http-server --cors` 启动后，可配合 `fetch` 或 `XMLHttpRequest` 正常请求同服务器下的资源。

3. **模拟 CDN 或生产环境**  
   使用 `-c 3600` 模拟缓存头，测试强缓存效果；或用 `-S` 测试 HTTPS 下的行为。

4. **局域网临时共享文件**  
   在同一网络下的设备（手机、另一台电脑）上，访问启动时打印的 IP 地址即可下载或预览文件，省去搭建 FTP 或 SMB 的麻烦。

5. **SPA 路由支持**  
   虽然 `http-server` 本身不处理前端路由重定向，但可配合 `--proxy` 将未匹配的请求代理到入口页面（需搭配简单代理逻辑），或改用 `serve`、`live-server` 等更专业的工具。

6. **自动化测试的静态服务器**  
   在 CI/CD 或 E2E 测试（如 Cypress、Playwright）中，用 `http-server` 快速起一个服务供测试用例访问，无需复杂配置。

### 五、注意事项

- **仅用于开发/测试环境**：生产环境应使用 Nginx、Apache 或专门的 Node 服务（如 Express、Koa）以获得更好的性能、安全和稳定性。
- **无实时重载**：修改文件后需手动刷新页面。需要自动刷新的可使用 `live-server`。
- **目录遍历默认开启**：若不想暴露整个目录结构，启动时加 `-d false`。
