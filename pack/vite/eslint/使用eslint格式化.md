你可以直接使用 ESLint 来格式化 JavaScript 文件。

ESLint 的核心功能是**代码检查**，用于发现代码中的潜在错误和不规范的写法。但它也具备强大的**自动修复**能力，可以一键修复许多格式问题，从而达到格式化的效果。

### 🚀 如何使用 ESLint 格式化代码

使用 ESLint 格式化代码非常简单，主要通过 `--fix` 参数来实现。

1.  **在项目根目录打开终端。**
2.  **执行格式化命令。**

    *   **格式化整个项目**：
        这条命令会递归地查找并尝试修复当前目录下所有 ESLint 能识别的文件。
        ```bash
        npx eslint . --fix
        ```

    *   **格式化指定的单个文件**：
        ```bash
        npx eslint your-file.js --fix
        ```

    *   **格式化指定目录下的特定类型文件**：
        例如，只格式化 `src` 目录下的所有 `.js` 文件。
        ```bash
        npx eslint "src/**/*.js" --fix
        ```

### ✨ 核心要点

*   **修复范围**：`eslint --fix` 只能自动修复那些支持自动修复的规则。
    *   **可以修复**：缩进 (`indent`)、引号 (`quotes`)、分号 (`semi`)、行尾空格 (`no-trailing-spaces`) 等格式问题。
    *   **无法修复**：未使用的变量 (`no-unused-vars`)、未定义的变量 (`no-undef`) 等逻辑错误。

*   **配置文件**：ESLint 的行为由项目根目录下的配置文件（如 `.eslintrc.js`）决定。你可以根据自己的项目需求定制代码规则，`--fix` 命令会严格按照这些规则来修复代码。

### 💡 最佳实践：与 Prettier 配合

在实际开发中，通常会将 ESLint 与 **Prettier** 结合使用，以实现代码质量和格式的完美统一。

*   **分工明确**：ESLint 负责检查代码质量和逻辑错误，而 Prettier 专注于纯粹的代码风格美化。
*   **避免冲突**：通过特定的配置（如使用 `eslint-config-prettier`），可以禁用 ESLint 中与 Prettier 冲突的格式化规则，让两者和谐共存。
*   **统一入口**：配置完成后，你依然可以使用 `eslint --fix` 命令，它会在检查代码质量的同时，调用 Prettier 来格式化代码，一举两得。
