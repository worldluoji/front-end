# env
build 时会默认读取.evn.production，但敏感信息不要放到里面，应该在云平台的环境变量中进行配置。

还有一个主意点，Next.js 约定：所有以 NEXT_PUBLIC_ 开头的环境变量，会在构建阶段（next build）被硬编码到客户端 JavaScript 中。也就是说，在浏览器端运行的代码里，process.env.NEXT_PUBLIC_TCB_ENV_ID 会被直接替换成一个字符串常量，而不是真的去读取系统环境变量。

任何以 NEXT_PUBLIC_ 开头的环境变量，最终都会暴露在浏览器端 JavaScript 中，任何人都可以在浏览器的开发者工具里看到。 因此，绝对不能用这个前缀存放敏感信息。
