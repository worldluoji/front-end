# Node.js 事件驱动模型​ 🦌
一个高效的服务员（事件循环）管理所有顾客, 服务员不等待，而是记录订单后立即服务下个顾客,菜品准备好时（I/O完成），通知顾客取餐。

[点餐示例](./examples/order.js)

---

理解六个阶段： [六个阶段](./examples/stage.mjs)

事件循环不是真正的"循环"，而是一个精心设计的调度器。它确保：
- 高吞吐量​ - 避免线程上下文切换的开销
- 低延迟​ - 快速响应I/O完成事件
- 资源高效​ - 单线程处理成千上万的并发连接
